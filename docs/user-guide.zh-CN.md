# SightFlow Desktop Agent 用户使用说明书

SightFlow Desktop Agent 是一个桌面聊天智能体。它可以观察指定聊天窗口中的新消息，调用模型分析聊天内容，并把生成的回复发送到当前聊天窗口。

使用前请先确认：

- 你已经准备好了一个支持 OpenAI 兼容接口的 API Key
- 电脑上已经打开了要使用的聊天软件
- 你愿意先在测试会话中完成一次完整联调

## 1. 适用场景

SightFlow 适合用于：

- 微信或企业微信的桌面聊天辅助回复
- 单会话、低风险场景下的客服辅助
- 通过视觉模型识别聊天窗口，再生成回复内容
- 通过本地控制接口让其他工具启动或暂停智能体

不建议直接用于：

- 未经确认的大量自动群发
- 涉及法律、医疗、财务、隐私等高风险内容的全自动回复
- 没有人复核的正式客服生产环境

## 2. 首次打开应用

启动 SightFlow 后，你会看到主界面。主界面主要包含：

- 当前状态：显示智能体是否正在运行
- 目标应用：选择要辅助的聊天软件
- 运行日志：显示识别、分析、跳过、回复等记录
- 启动引擎：开始运行智能体
- 停止引擎：停止运行智能体
- 设置：打开配置窗口

首次使用时，不要直接点击启动。先完成基础配置和目标应用设置。

## 3. 配置基础参数

1. 点击主界面右下角的设置按钮。
2. 进入「基础配置」。
3. 在「视觉配置」中填写：
   - 视觉接口密钥
   - 视觉模型
   - 视觉服务地址
4. 在「回复模型配置」中填写：
   - 回复模型 API Key
   - 回复模型
   - 回复服务地址
5. 点击「测试连接」。
6. 点击「拉取模型」或「刷新列表」。
7. 点击「保存配置」。

基础配置分成两类：

- 视觉配置：用于识别聊天窗口、输入框、未读消息等位置
- 回复模型配置：用于驱动智能体分析聊天内容并生成回复

视觉模型和回复模型可以使用同一套 API Key、模型和 Base URL，也可以分别使用不同模型服务。

## 4. Base URL 写法

`Base URL` 支持两种写法：

- 根地址，例如 `https://your-gateway.example/v1`
- 完整聊天接口地址，例如 `https://your-gateway.example/v1/chat/completions`

应用会自动兼容这两种格式。

## 5. 配置自检清单

启动前建议按下面顺序自检一次：

1. 在「视觉配置」里确认已经填写：
   - 视觉接口密钥
   - 视觉模型
   - 视觉服务地址
2. 在「回复模型配置」里确认：
   - 已单独填写回复配置
   - 或已经和视觉配置同步
3. 点击「测试连接」和「拉取模型 / 刷新列表」。
4. 查看中间区域的「最近诊断」：
   - 连接测试是否成功
   - 拉取模型是否成功
   - 是否出现失败分类和下一步建议
5. 查看右侧的「Service Health」：
   - 当前是“连接成功”“列表失败”“已恢复缓存”还是“待测试”
   - 最近一次成功发生在什么时间
6. 查看右侧的「Startup Preflight」：
   - 如果显示“可启动”，说明当前配置可以通过启动前检查
   - 如果显示“启动前拦截”，优先按提示修正配置

## 6. 回复配置与视觉配置的关系

回复模型配置有三种状态：

- 继承视觉配置：回复模型暂时复用视觉模型的 Key、模型和 Base URL
- 已同步视觉配置：你已经把视觉模型配置显式同步到回复模型配置
- 独立配置：回复模型使用自己单独的 Key、模型或 Base URL

如果你只配置了视觉模型，但还没有单独配置回复模型，可以在「回复模型配置」里点击「同步视觉配置」，把当前视觉模型的 Key、模型和地址一键写入回复模型配置。

同步完成后，回复配置会真正保存到本机配置文件中，后续启动不会再依赖临时继承。

## 7. 诊断按钮说明

设置页右侧和中间区域提供了几种诊断工具：

- 测试连接：验证当前模型接口能否成功调用
- 拉取模型 / 刷新列表：从当前模型服务拉取可用模型列表
- 清缓存：清空本地保存的模型候选列表，再重新拉取
- 导出诊断：复制完整模型接入诊断信息
- 导出失败：只复制当前失败项，适合直接发给技术支持或服务商
- 重置诊断：清空当前诊断结果和最近成功记录
- 重新检查：重新运行启动前检查，不启动引擎、不操作聊天软件

## 8. 选择智能体

进入设置窗口的「智能体」页面。

默认会提供内置的「豆包 Seed」智能体。普通用户可以直接使用默认智能体，它会使用「回复模型配置」中的 API Key、模型和 Base URL。

如果你使用的是第三方智能体，需要根据服务提供方提供的说明填写配置。

## 9. 选择目标应用

回到主界面，在「目标应用」中选择要使用的聊天软件。

### 微信 / 企业微信

微信和企业微信默认优先使用自动视觉识别。

使用前请确认：

- 已经打开微信或企业微信
- 聊天窗口没有最小化
- 聊天窗口没有被其他窗口完全遮挡
- 网络可以正常访问模型服务

### 钉钉 / 飞书 / Slack / Telegram / 其他桌面应用

这些应用默认使用手动框选模式。

首次使用时需要点击「开始框选」，依次圈出：

1. 会话列表
2. 聊天内容区
3. 输入框

框选完成后，应用会保存这组区域。以后再次选择同一个目标应用时，会自动复用上一次的框选结果。

## 10. 启动智能体

完成配置后：

1. 打开目标聊天软件
2. 进入希望处理的聊天窗口
3. 回到 SightFlow 主界面
4. 点击「启动引擎」

启动后，运行日志会显示当前进度，例如：

- 正在识别聊天窗口布局
- 聊天窗口识别完成
- 正在分析聊天内容
- 本轮无需回复
- 检测到当前对话有新消息
- 已生成回复

如果运行正常，状态会变成「运行中」。

## 11. 停止智能体

需要暂停自动处理时，点击主界面的「停止引擎」。

停止后：

- 智能体不会继续观察聊天变化
- 不会继续调用模型生成回复
- 不会继续发送消息

再次使用时，点击「启动引擎」即可。

## 12. 自动回复逻辑说明

SightFlow 的基本流程是：

1. 识别聊天窗口布局
2. 截取聊天内容区域
3. 把截图交给智能体分析
4. 判断是否需要回复
5. 生成回复文本
6. 把文本发送到输入框或直接发送
7. 等待下一轮变化

内置智能体会尽量避免自我循环回复。例如，当模型判断最后一条消息更像是你自己发出的内容时，通常会选择跳过。

## 13. 常见问题

### 启动时提示未填写视觉接口密钥

原因：当前目标应用会使用自动视觉识别，但视觉配置里没有可用 API Key。

处理方式：

1. 打开设置
2. 进入「基础配置」
3. 在「视觉配置」中填写视觉接口密钥、模型和服务地址
4. 保存配置

### 启动时提示未填写回复模型 API Key

原因：当前智能体需要回复模型配置，但回复模型没有可用 Key。

处理方式：

1. 打开设置
2. 进入「基础配置」
3. 填写回复模型配置，或者点击「同步视觉配置」
4. 测试连接
5. 保存配置

### 测试连接失败

常见原因：

- API Key 填错
- 当前网络无法访问模型服务
- API Key 权限不足
- Base URL 填错
- 模型名填错

处理方式：

- 重新复制 API Key
- 检查网络连接
- 检查 Base URL 是否为兼容接口根路径
- 刷新模型列表后重新选择模型
- 查看失败分类和下一步建议
- 使用「导出失败」把关键信息发给技术支持或服务商

### 框选模式下没有反应

常见原因：

- 聊天内容区框选不准确
- 输入框框选不准确
- 聊天窗口位置或大小发生变化
- 当前会话没有新消息

处理方式：

- 点击「重新框选」
- 重新圈出会话列表、聊天内容区和输入框
- 保持聊天窗口位置稳定

### 发送到了错误位置

请立即点击「停止引擎」，然后检查：

- 是否选错目标应用
- 是否框选了错误输入框
- 聊天窗口是否被移动或缩放
- 当前是否切换到了别的会话

## 14. 本地控制接口

如果你使用支持本地工具调用的平台，可以通过本地接口启动或暂停 SightFlow。

接口只监听本地地址：

```text
127.0.0.1
```

常用接口：

- 查询状态：`GET http://127.0.0.1:12680/skill/status`
- 启动智能体：`POST http://127.0.0.1:12680/skill/start`
- 暂停智能体：`POST http://127.0.0.1:12680/skill/pause`

注意：这些接口只有在 SightFlow 应用已经打开时才可用。
