# SightFlow 商用试点常见问题 FAQ

本文档汇总试点第一周最容易被客服和实施工程师问到的问题。

## 1. 账号、授权、试用

Q1.1:一台机器可以多个客服同时登录用 SightFlow 吗?
A:不行,SightFlow 是单机应用,一个 Windows 用户一个授权,不支持多客服同时挂在同一台机器。

Q1.2:授权到期了会自动停止吗?
A:会。授权到期时,自动发送会被熔断,只保留草稿模式;引擎不会自动停止,你可以看到草稿,等续期后再发。

Q1.3:试用 14 天是从安装算还是从第一次启动算?
A:从第一次启动算。%APPDATA%\SightFlow\license.json 里的 activatedAt 字段就是起点。

Q1.4:换电脑了,试用次数会重置吗?
A:不会,SightFlow 用机器码 + 操作系统用户名做唯一性判断。换电脑需要重新申请试用或商用授权。

Q1.5:商用授权可以一台机器多份吗?
A:每台机器一份,不能多份共用。机器码绑定后,授权文件 %APPDATA%\SightFlow\license.json 就不能直接复制到另一台机器。

## 2. 自动回复行为

Q2.1:SightFlow 会回复我自己发的消息吗?
A:不会。识别阶段会用消息方向和发送者字段判断,自己发的消息会被跳过,在 Console 出现一条 skipped,原因是 own-message。

Q2.2:客户发了一张图片,会自动回复吗?
A:会进入识别流程,但默认不会回复图片内容,只会用通用话术让客户补文字。可以在控制台 → 知识库里给"图片消息"配专门的回复模板。

Q2.3:群里没有被 @ 会被自动回复吗?
A:默认不会。群聊只在以下情况进入自动回复:被 @、命中白名单关键词、手动触发。

Q2.4:含"投诉""退款""金额"的消息会被回复吗?
A:不会,会被 ReplyPolicy 拦下,Console 出现 blocked,原因含 high-risk。这一类问题必须人工接管。

Q2.5:SightFlow 会连续发同一条消息吗?
A:有去重保护。同一条 customer message 24 小时内不会触发第二次自动发送;连续回复同一客户会在控制台里合并提示,防止刷屏。

## 3. 性能与稳定性

Q3.1:SightFlow 在后台一直开会不会很占 CPU?
A:空闲时 CPU 占用 < 1%(单核),主要消耗在定时扫描;只有识别 + 调用模型时会短时间涨到 5%-15%。

Q3.2:能连续开 8 小时吗?
A:可以。试点要求是连续 8 小时不崩溃。如果出现崩溃,会自动写入主进程日志,在 Console → 诊断面板可以导出。

Q3.3:用着发现内存一直涨,怎么排查?
A:Console → 诊断 → 导出诊断包,把 .zip 发给工程。诊断包里包含内存快照、关键截图、traces,工程侧会用 memgraph 复现。

## 4. 知识库与回复质量

Q4.1:知识库支持哪些格式?
A:JSON 数组和 CSV(带表头)。每个条目至少包含 question 和 answer 字段,可选 category / tags / priority。

Q4.2:知识库导入后多久生效?
A:立即生效。导入后会重建索引,索引大小一般远小于原文件;如果导入 > 5000 条,首次重建会需要几秒,Console 会显示加载状态。

Q4.3:知识库没命中时 SightFlow 怎么回?
A:会用"我不确定,可以换个问法吗"这类保守话术,同时把这次未命中的 question 写入草稿,等客服补充后回流到知识库。

Q4.4:能加品牌语气吗?
A:可以。控制台 → 知识库 → 品牌语气,填入"亲切 / 简洁 / 不用表情包"等指令,所有回复都会带上。

Q4.5:同一问题不同客户反复问,怎么避免重复?
A:在知识库里给高频问题加 question_alias 字段,系统会做语义归一化,只回一次标准答案。

## 5. 审计与可观测

Q5.1:我能回看昨晚某条自动回复为什么这么回吗?
A:可以。Console → 会话记录 → 选 trace → 看 trace 详情,包含:截图、识别结果、模型输入摘要、策略判断、发送结果、耗时。

Q5.2:导出诊断包会包含客户隐私吗?
A:会。诊断包包含截图、识别文本、trace 明文。导出前请确认已脱敏或已征得客户同意;试点期间建议诊断包只在工程内网流转。

Q5.3:客户能直接看到 SightFlow 在跑吗?
A:不能。SightFlow 在客户侧的聊天窗口里看起来和人工回复完全一样,不会在消息里带标记。如果客户合规要求标记,需要在合同里约定。

## 6. 集成与扩展

Q6.1:SightFlow 能接企业自己的 CRM / 工单系统吗?
A:目前只支持单向:把会话 trace 导出 CSV,导入 CRM。不支持反向把 CRM 客户信息回灌到 SightFlow。这一项在第 13 周之后规划。

Q6.2:能不能只让 SightFlow 处理特定客户?
A:可以。控制台 → 自动化范围 → 客户白名单,加进去的客户才进入自动回复链路;黑名单客户永远跳过。

Q6.3:可以只对部分会话开自动回复吗?
A:可以。控制台 → 自动化范围 → 会话白名单,加进去的会话名才自动回复;其它会话只生成草稿。

Q6.4:能不能禁用某些关键词,一旦客户提到就直接转人工?
A:可以。控制台 → ReplyPolicy → 敏感词,加入"投诉""律师函""12315"等词,命中后直接 blocked + 建议人工。

## 7. 升级、迁移、卸载

Q7.1:升级版本会丢数据吗?
A:不会。所有用户数据都在 %APPDATA%\SightFlow,升级安装器只替换程序文件,数据原封不动。

Q7.2:从 1.0.0 升到 1.1.0 需要重新激活吗?
A:不需要。授权和机器码不变,只更新程序文件。

Q7.3:想把所有客服电脑升级到统一版本,怎么做?
A:用 NSIS 安装器静默安装参数 /S 可以批量推;也可以用配置导入导出功能,先把基础配置和知识库在 1 台机器上准备好,再分发给其它机器。

Q7.4:卸载了想重装,授权还有效吗?
A:有效,但要先确认 license.json 没被一起删掉。如果删了,需要重新申请。

## 8. 紧急情况

Q8.1:SightFlow 给客户回错了,马上要撤回,怎么办?
A:SightFlow 不直接撤回消息。第一时间:
1. 在 Console 把出错会话标记为"人工接管",后续动作由人工接管。
2. 客服在微信 / 企微侧手动撤回错误消息(2 分钟内可撤回)。
3. 在工单里写明 trace id 和原因,工程团队会从 Console 拉出当时的截图、识别、决策、发送全链路。

Q8.2:SightFlow 反复给同一客户发同样的话,怎么紧急停?
A:
1. 主界面点"停止引擎"。
2. Console → 自动化范围 → 把该客户加入黑名单。
3. 导出诊断包,保留证据。
4. 等工程团队定位:通常是模型降级导致生成文本相似度过高,或输入框粘贴 fallback 失败。

Q8.3:发现 SightFlow 在不该自动回复的会话里发了消息,怎么补救?
A:
1. 立刻点"停止引擎"。
2. 在 Console 把所有相关会话都设为"人工接管"。
3. 检查"自动化范围"配置,确认白名单和黑名单都设置正确。
4. 导出诊断包,工程团队会复盘为什么被错误地纳入了自动发送范围。

## 9. 与其他产品的区别

Q9.1:SightFlow 和"微信机器人协议版"工具有什么区别?
A:SightFlow 不接微信协议、不做 Hook、不注入数据库。试点走视觉 + UIAutomation + RPA,合规风险低,被封号概率几乎为零。协议版功能更强但封号风险高,通常不适合商用客服场景。

Q9.2:SightFlow 和普通的聊天机器人 SaaS 有什么不一样?
A:普通 SaaS 需要把客户的微信账号托管到云端;SightFlow 完全跑在客户本地电脑上,所有数据不外发,模型和知识库都可以走客户自己的私有部署。

Q9.3:试点结束后,SightFlow 的商用版大概定价?
A:商务侧还在定,大致方向是:按机器数 + 客服人数 + 是否私有部署三档。具体需要联系商务。

## 10. 反馈与改进

Q10.1:用着有想法或建议,怎么提?
A:Console → 诊断 → 导出诊断包,在工单里附上建议。试点期间所有反馈 24 小时内会有工程团队回复。

Q10.2:发现 bug,怎么报?
A:Console → 诊断 → 导出诊断包,在工单里附上 trace id + 现象截图 + 复现步骤。

Q10.3:能远程接入排查吗?
A:可以。试点期间工程团队通过商用 SSH 跳板接入,接入前会通知客服并保留操作日志,排查完成后立即退出。

如果遇到本文档没有覆盖的问题,请第一时间按以下顺序处理:
1. 主界面点"停止引擎",先停再说。
2. Console → 诊断 → 导出诊断包。
3. 联系工程团队 oncall,带上诊断包和现象描述。
