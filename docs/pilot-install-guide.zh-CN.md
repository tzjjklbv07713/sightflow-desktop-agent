# SightFlow 商用试点安装与首次配置指南

本指南面向第一次在一台干净的 Windows 客服电脑上部署 SightFlow 商用版的实施工程师。请按顺序操作,不要跳步。

版本基线:

- SightFlow 1.0.0
- Electron 32 + Node 20 LTS + electron-vite
- Windows 10 22H2 及以上 / Windows 11
- 仅支持 Windows 微信 3.9.x 系列与 Windows 企业微信 4.x 系列

## 1. 部署前检查

在目标机器上先确认以下条目,不通过的机器不要继续:

1. 操作系统是 64 位 Windows 10 或 11,专业版/企业版优先。
2. 已经退出所有残留的 微信 / 企业微信 / SightFlow 进程,任务管理器里没有 WeChat.exe / WXWork.exe / SightFlow.exe。
3. 目标 IM 客户端能正常登录并看到消息,屏幕缩放设置保持 100% 或 125%(不要使用 150% 以上的缩放,会显著降低识别准确率)。
4. 机器可以访问你的视觉模型服务、回复模型服务以及授权服务地址(出网白名单已开)。
5. C 盘剩余空间大于等于 5 GB,用户目录(C:\Users\<用户>)可写。
6. 关闭所有杀毒软件对 SightFlow 安装目录的实时扫描,否则 UIAutomation 会被拦截。

## 2. 安装 IM 客户端

按客户实际使用的 IM 选择:

- 微信:安装 微信 for Windows 3.9.6 及以上,固定窗口位置到屏幕左上或右上,关闭自动更新以外的托盘常驻弹窗。
- 企业微信:安装 Windows 企微 4.0.6 及以上,登录后保持主窗口置顶,关闭消息预览浮窗。

两种 IM 都不要使用绿色版、第三方修改版或企业定制版。

## 3. 安装 SightFlow

### 3.1 通过 NSIS 安装器(推荐)

1. 拿到 SightFlow-1.0.0-setup.exe 后右键以管理员身份运行。
2. 安装路径建议保持默认 C:\Program Files\SightFlow,不要改到桌面、下载目录、U 盘。
3. 安装器会创建桌面快捷方式和开始菜单项。
4. 安装完成后,先不要启动 SightFlow,先回到第 2 节确认 IM 已经登录。

### 3.2 通过解压版(仅供内部调试)

    npm ci
    npm run build
    npm run build:unpack

解压版产物在 dist/win-unpacked/SightFlow.exe,双击即可运行,但不会有开机自启、卸载程序和注册表项,不适合交付给客户。

## 4. 首次启动与初始化向导

启动 SightFlow 后,如果客户是第一次使用,会自动弹出初始化向导,分四步:

1. 选择 IM 类型:微信 / 企业微信。选错可以在控制台首页点重新初始化重选。
2. 配置回复模型:填入回复模型 Base URL、API Key、模型名,点测试连接。
3. 导入知识库:选择一份预先生成的 JSON / CSV 知识库文件,导入后会在控制台「知识库」面板看到条目数量。
4. 烟雾测试:在一个测试会话里点发送一条测试消息,确认识别 → 回复 → 发送的链路跑通。

完成后向导会写入 onboardingCompletedAt,以后启动不再弹出。
如果中途某一步失败,可以关闭窗口再开,SightFlow 会从第一步重新走。

## 5. 授权激活

初始化完成后,需要激活授权才能进入商用模式。

### 5.1 试用授权

不填激活码,直接保存,SightFlow 会进入 14 天试用。试用期间:

- 全部功能可用
- 启动时会在状态栏显示试用还剩 X 天
- 试用到期后,自动发送会被熔断,只保留草稿模式

### 5.2 商用授权

1. 拿到形如 SF-COM-XXXX-XXXX-XXXX 的授权码。
2. 打开控制台 →「授权」面板。
3. 粘贴授权码,点击激活。
4. 激活成功会显示:计划=商用、机器码=服务端登记值、到期时间=激活后 366 天。
5. 授权文件保存在 %APPDATA%\SightFlow\license.json,不要手动修改。

授权与机器绑定,换主板或重装系统需要重新申请授权码。

## 6. 配置导入与导出

为多台客服电脑复制部署,SightFlow 提供了配置导入导出。

- 导出:控制台 →「设置」面板 → 导出当前配置,会生成一份 sightflow-settings-*.json,包含基础配置、回复策略、自动化范围,但不包含授权码和知识库内容。
- 导入:在新机器上,控制台 →「设置」面板 → 导入配置,选择刚才那份 JSON,导入后会要求重启。

注意:导入配置会覆盖当前所有设置,请提前导出当前配置作为备份。

## 7. 最小冒烟用例

完成以上步骤后,建议用以下用例做一次最小冒烟,确保商用模式可用:

| 用例 | 预期 |
| --- | --- |
| 收到一条普通咨询 | 5 秒内回复,Console 出现一条 sent 记录 |
| 自己刚发出一条消息 | 不触发自动回复,Console 出现一条 skipped 记录 |
| 收到一条含投诉的消息 | 不自动回复,Console 出现一条 blocked 记录,原因含 high-risk |
| 收到一条群消息且未被 @ | 不自动回复,Console 出现一条 skipped 记录,原因含 group-not-mention |
| 关闭 IM 主窗口 | 引擎自动暂停,Console 出现一条 warning |
| 连续点击 5 次启动引擎 | 第二次以后被熔断,只更新状态不重复启动 |

冒烟用例全部通过后,可以让客服同学开始使用。

## 8. 安装失败的常见回退

| 现象 | 可能原因 | 下一步 |
| --- | --- | --- |
| 安装器报 2503 / 2502 | 没有以管理员身份运行 | 重新右键以管理员身份运行 |
| 启动后白屏 | electron 渲染进程崩溃 | 删 %APPDATA%\SightFlow 后重启;若仍异常,联系工程 |
| 视觉模型测试连接失败 | 网络或代理 | 在控制台 →「设置」里检查 Base URL,确认能 curl 通 |
| 授权激活失败 | 机器码与服务端登记不一致 | 把控制台 →「授权」面板显示的机器码发给商务核对 |
| 知识库导入无反应 | 文件不是 JSON / CSV | 打开文件确认是合法 JSON 数组或带表头的 CSV |
