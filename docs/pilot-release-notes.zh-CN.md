# SightFlow 1.0.0 商用试点发布说明

发布时间:2026-06-23
目标客户:第一期商用试点(微信 / 企业微信 私域客服)
适用版本:SightFlow 1.0.0 (Windows 64 位)

## 1. 本版本包含什么

- 视觉 + UIAutomation + RPA 的桌面识别引擎,支持 Windows 微信 3.9.x 和 Windows 企业微信 4.x。
- 第一次启动时弹出初始化向导,引导客服完成 IM 选择、模型配置、知识库导入、烟雾测试。
- 商用控制台(Console),包含 6 个面板:总览 / 会话记录 / Trace 详情 / 知识库 / 授权 / 诊断。
- 自动发送安全闸门:敏感词拦截、群聊降权、人工接管、连续失败熔断、授权到期熔断。
- 本地授权服务:SF-COM- 前缀为商用(366 天),其他为试用(14 天)。
- 配置导入导出:把基础配置和回复策略在多台客服电脑之间复制。
- 诊断导出包:把配置摘要、错误日志、关键截图、运行指标打包成 zip,方便远程排障。

## 2. 不包含什么

- macOS / Linux 商用支持(本版本只验证 Windows)。
- 微信协议接入、Hook、数据库注入(我们不走那条路线)。
- 群机器人场景自动化(群聊默认不自动回复,只生成草稿)。
- 多客服协作 / 工单系统双向同步(本版本只支持单向 trace 导出)。
- 国际 IM(WhatsApp / Telegram / Messenger)支持。

## 5. 已知问题 / 限制

- 屏幕缩放 150% 及以上,UIAutomation 准确率会显著下降;试点要求客服电脑缩放保持 100% 或 125%。
- 微信 / 企微 大版本更新后,布局检测可能失败;需要重新走初始化向导。
- 视觉模型服务的可用性直接决定自动回复质量,试点客户必须保证模型服务 99% 可用。
- 商用授权与机器码绑定,主板更换或重装系统需要重新申请。

## 6. 试点观察指标

试点期间工程团队会持续收集以下指标,异常时会主动联系客户:

- 单台客服机连续运行 8 小时的崩溃次数(目标:0)
- 自动回复平均响应时长(目标:小于 5 秒)
- 高风险消息被自动发送的次数(目标:0)
- 群聊误回复次数(目标:0)
- 客服对自动回复结果的确认/修改比例(用于评估回复质量)
- 诊断包导出次数与平均解决时长

## 7. 升级与回滚路径

升级:
1. 在 Windows 设置 -> 应用里点 SightFlow -> 卸载。
2. 运行新版本安装器。
3. 升级不影响 %APPDATA%\SightFlow 下的设置、授权、trace、知识库。

回滚:
1. 等级 A:主界面点停止引擎。
2. 等级 B:关闭 SightFlow、移除启动快捷方式、导出诊断包。
3. 等级 C:卸载程序 + 删除 %APPDATA%\SightFlow。
4. 详细步骤见 pilot-rollback.zh-CN.md。

## 8. 反馈渠道

试用过程中任何问题或建议,请按以下方式反馈:

- Console -> 诊断 -> 导出诊断包,把 zip 附件发到工程 oncall 邮箱(以商务侧确认为准)。
- 工单里附上:trace id、现象描述、复现步骤、影响客户数。
- 试点期间工程团队 oncall 7x24 小时响应,P0 工单 2 小时内必须回复,P1 工单当天回复。
